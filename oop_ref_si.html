Final Reflection & Learning Outcomes

    Looking a bit into testing and using this as a concluding reflection on system implementation (because testing goes till the end of your code), I fairly have to say that in the past (before working with unittest in Python), I also used a testing strategy which included different checkpoints. In this sense, one test was a bit similar to unittest so that I simply used one if-else-statement and if the statement turned out tu be true then I printed out that the check was successful ("Check: OK") and similarly for the case when the test failed ("Check: False"). In this perspective, I realized some days ago what the unittest library can do for you. It simply gives you a "framework" for running tests ("OK" for true, "F" for false and "E" for errors). You can put into your test class whatever you want to test and the unittest library surrounds it then with the usual outputs (OK,F or E) (Philips, 2018).

    What I have learnt and how?

  • System Implementation in Using OOP
  • How to Connect a Database to a Python file
  • Connection between Database, Logic File (for example Python) and a User Interface
  • Understanding Trade-Off's of different API Architectures while Developping an Application
  • Working with the Unittest Library in Python for Testing
